В данном случае план выполнения запроса состоит из операции Nested Loop (вложенных циклов). Во время выполнения операции Nested Loop происходит выполнение двух операций-источников данных.

Первая операция - Seq Scan on "Н_ОЦЕНКИ" - выполняется путем прохода по всей таблице "Н_ОЦЕНКИ" (Seq Scan), где происходит фильтрация по условию "КОД = 'неявка'", и выбирается 1 строка (rows=1).

Вторая операция - Bitmap Heap Scan on "Н_ВЕДОМОСТИ". При этом происходит проверка условий "ЧЛВК_ИД < 142390", "ЧЛВК_ИД = 153285" и "ОЦЕНКА = 'неявка'". 



По плану выполнения видно следующее:
1. Первым шагом происходит выполнение операции Hash Right Join. 

2. Внутри операции Hash Right Join происходит выполнение двух операций:

- Операция Seq Scan на таблице "Н_УЧЕНИКИ". Она выполняется путем прохода по всей таблице "Н_УЧЕНИКИ" (Seq Scan).
- Операция Hash на таблице "Н_ЛЮДИ". Она выполняется для создания хеш-таблицы с хешированными значениями столбца "ИД" таблицы "Н_ЛЮДИ".

3. Внутри операции Hash Right Join также выполняется операция Hash Join. В этой операции происходит объединение двух таблиц "Н_ЛЮДИ" и "Н_ОБУЧЕНИЯ" по условию "Н_ЛЮДИ"."ИД" = "Н_ОБУЧЕНИЯ"."ЧЛВК_ИД".

- Операция Seq Scan на таблице "Н_ЛЮДИ" выполняется для прохода по всей таблице и получения строк, удовлетворяющих условию ("ИМЯ")::text > 'Александр'::text.
